# A.1 用語一覧

本書で頻繁に登場した用語を整理する。

| 用語                | 説明                                                                |
| ------------------- | ------------------------------------------------------------------- |
| Virtual File System | メモリ上に構築される仮想ファイルシステム                            |
| System              | TypeScript が利用するファイルシステム抽象化インターフェース         |
| CompilerHost        | Compiler がファイルへアクセスするためのインターフェース             |
| LanguageServiceHost | Language Service がプロジェクト情報を取得するためのインターフェース |
| Language Service    | コード補完や型情報取得などを担当するサービス                        |
| Environment         | TypeScript 実行環境全体を管理するオブジェクト                       |
| Program             | コンパイル対象となるプロジェクト全体を表現するオブジェクト          |
| SourceFile          | 解析済みの 1 ファイルを表すオブジェクト                             |
| Emit                | JavaScript や Declaration File を生成する処理                       |

# A.2 主要 API 一覧

`@typescript/vfs`

で利用頻度の高い公開 API をまとめる。

| API                                  | 主な役割                                         |
| ------------------------------------ | ------------------------------------------------ |
| createDefaultMapFromNodeModules()    | 標準ライブラリを含む初期 Map を生成する          |
| createSystem()                       | Map を `ts.System` へ変換する                    |
| createVirtualCompilerHost()          | Virtual File System 用の CompilerHost を生成する |
| createVirtualTypeScriptEnvironment() | Environment を生成する                           |
| updateFile()                         | ファイル内容を更新する                           |
| createFile()                         | 新しいファイルを追加する                         |
| deleteFile()                         | ファイルを削除する                               |
| emit()                               | JavaScript を生成する                            |

これらは、役割ごとに独立した Factory Function または操作 API として設計されている。

# A.3 オブジェクトの依存関係

本書全体で説明してきた依存関係を、一枚の図にまとめる。

```text
                    Map
                     │
                     ▼
                 ts.System

                     │
                     ▼
              CompilerHost
                     │
                     ▼
          LanguageServiceHost
                     │
                     ▼
            Language Service
                     │
                     ▼
 VirtualTypeScriptEnvironment
```

実際には、Environment が Language Service や Host を保持しているため、オブジェクトの所有関係は逆方向になる部分もある。

しかし、初期化の流れとして理解するなら、この図が最も分かりやすい。

# A.4 TypeScript Compiler API との対応

`@typescript/vfs` 自身は、Compiler を実装しているわけではない。

TypeScript Compiler API の各機能を、Virtual File System 上でも利用できるようにしている。

対応関係をまとめると、次のようになる。

| @typescript/vfs              | TypeScript Compiler API           |
| ---------------------------- | --------------------------------- |
| Virtual File System          | ts.System                         |
| createSystem()               | ts.System                         |
| createVirtualCompilerHost()  | ts.CompilerHost                   |
| VirtualTypeScriptEnvironment | ts.LanguageService                |
| emit()                       | Program.emit()                    |
| updateFile()                 | Language Service の更新処理を補助 |

重要なのは、VFS が Compiler API を置き換えるものではない という点である。

# A.7 レイヤーごとの責務

本書では、「責務を分離する」 という設計思想が何度も登場した。

改めて、各レイヤーが担当している役割を整理する。

| レイヤー                     | 主な責務                                           |
| ---------------------------- | -------------------------------------------------- |
| Map                          | ファイル内容を保持する                             |
| Virtual File System          | 仮想ファイルシステムとして管理する                 |
| System                       | TypeScript が利用できるファイル操作 API を提供する |
| CompilerHost                 | Compiler にファイル情報を提供する                  |
| LanguageServiceHost          | Language Service にプロジェクト情報を提供する      |
| Language Service             | 型解析・補完・定義ジャンプなどを提供する           |
| VirtualTypeScriptEnvironment | 各レイヤーをまとめて利用者へ公開する               |

このように、一つのレイヤーが 複数の責務を持たないよう設計されている。

# A.8 本書で登場した設計パターン

`@typescript/vfs` には、典型的な設計パターンがいくつか利用されている。

## Adapter

```text
Map
↓
System
```

Map を TypeScript が理解できる System へ変換する。

これは Adapter の代表例である。

---

## Factory

```text
createSystem()

createVirtualCompilerHost()

createVirtualTypeScriptEnvironment()
```

公開 API の多くは Factory Function として設計されている。

利用者は、複雑な初期化を意識する必要がない。

---

## Facade

VirtualTypeScriptEnvironment は、Language Service や CompilerHost などを内部に保持し、利用者へシンプルな API を提供する。

これは Facade の考え方に近い。

---

## Composition

Environment は、複数のオブジェクトを 継承ではなく、組み合わせによって構成している。

これは Composition を重視する 近年のライブラリ設計に共通する特徴である。

# A.9 よくある誤解

`@typescript/vfs` を初めて読むと、誤解しやすい点がいくつかある。

---

## 「VFS が型解析をしている」

誤りである。

型解析を担当しているのは、TypeScript Compiler API である。

VFS は、その実行環境を提供している。

---

## 「Map が Virtual File System」

厳密には異なる。

Map は Virtual File System の 内部データ構造である。

Virtual File System は、Map と、それを操作する仕組み全体を含む概念である。

---

## 「Environment が Compiler」

これも誤りである。

Environment は Compiler を保持しているわけではなく、Compiler API や Language Service を 利用しやすくまとめたオブジェクトである。

---

## 「ブラウザ専用ライブラリ」

これも正確ではない。

Virtual File System を利用することで、ブラウザでも動作しやすい設計になっているが、考え方そのものは Node.js など他の環境でも利用できる。

---

# A.10 ソースコードを読むときの着眼点

本書では、実装だけではなく、設計思想も重視して解説した。

実際にソースコードを読む際は、次のような点を意識すると理解しやすい。

---

## ① 何を実装しているか

まず、その関数が 何を担当しているのかを見る。

---

## ② 何を実装していないか

TypeScript 本体へ 処理を委譲している部分を探す。

VFS は 「自分で実装しない」 ことも重要な設計である。

---

## ③ どのレイヤーに属するか

例えば、

```text
Map
↓
System
↓
CompilerHost
↓
LanguageServiceHost
↓
Environment
```

の どこに位置するコードなのかを考える。

これだけで、責務が理解しやすくなる。

---

## ④ どこで責務を切り替えているか

設計の境界を意識する。

例えば、

- Map から System へ変換する瞬間
- CompilerHost を生成する瞬間
- Environment を返す瞬間

などは、レイヤーが切り替わる重要なポイントである。

本書では、`@typescript/vfs` を入口として、TypeScript Compiler API の構造を学んだ。

Virtual File System の構築から始まり、

- System
- CompilerHost
- LanguageServiceHost

そして

- VirtualTypeScriptEnvironment

へと順番に理解を深めてきた。

`@typescript/vfs` 自体は、数万行規模の巨大なライブラリではない。

しかし、その内部には、TypeScript Compiler API を理解するための 重要な考え方が凝縮されている。

責務を分離し、既存の Compiler API を再利用し、利用者へシンプルな API を提供する。

これが、`@typescript/vfs` の設計思想である。

本書を読み終えた時点で、読者は 単にライブラリの使い方を知っているだけではなく、TypeScript Compiler API 全体を学ぶための土台を身につけているはずである。

# おわりに

`@typescript/vfs` は、TypeScript Playground やブラウザ IDE を支えるための補助ライブラリとして開発されている。

しかし、その価値は、Virtual File System を提供することだけではない。

TypeScript Compiler API と利用者との橋渡しを行い、複雑な初期化や状態管理を隠蔽することで、ブラウザ上でも TypeScript の強力な機能を利用できる環境を実現している。

本書が、`@typescript/vfs` の理解だけでなく、TypeScript Compiler API 全体を学ぶための第一歩となれば幸いである。
